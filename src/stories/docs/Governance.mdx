# Governance

A design system becomes sustainable when ownership and change management are clear.

## Contribution flow

**Request → Review → Design → Build → Storybook QA → Release → Adoption**

## Definition of done

A shared component should include:

- Approved design specification
- Defined states and variants
- Storybook stories
- Controls for relevant inputs
- Accessibility review
- Responsive review
- Usage guidance
- CMS mapping when applicable

## Change types

### Patch
Small bug fix with no intended breaking change.

### Minor
Backward-compatible capability, variant, or enhancement.

### Major
Breaking API, visual, behavioural, or migration-impacting change.

## Ownership

Design and engineering jointly own component quality. Product and CMS teams provide usage feedback and adoption requirements.
