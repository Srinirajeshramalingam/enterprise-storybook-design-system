# Enterprise Design System

A scalable component system that connects **Figma design specifications**, **Storybook implementation**, and **CMS / AEM authoring patterns**.

## Purpose

This design system creates a shared source of truth across design, engineering, QA, and content teams.

### Core goals

- Create consistent digital experiences across products and websites.
- Reduce duplicate UI patterns and inconsistent implementations.
- Make component states, behaviour, accessibility, and usage visible.
- Support reusable CMS / AEM components and content authoring.
- Improve the design-to-development handoff.

## System model

**Design principles → Design tokens → Components → CMS patterns → Digital experiences**

## Delivery workflow

**Figma → Storybook → CMS / AEM → Website**

Storybook acts as the implementation and documentation layer between design specifications and production component delivery.

## Current component domains

- Actions
- Forms
- Data Display
- Feedback
- Navigation
- Overlays
- CMS Patterns
